Task
Build a recommendation system that, given a user's reading history, predicts what they should read next.
You are free to define and scope the problem — but be explicit about your choices. Some directions to consider:
●	Recommend the next chapter within all the books a user is currently reading
●	Recommend a new book to start based on reading history
●	Handle users or books with little to no history (cold start)
Dataset Overview
You are given two files. 
●	chapters.csv contains metadata for 50,000 chapters across ~50,000 books — each chapter belongs to a book, has a sequential position within it (chapter_sequence_no), an author, a publication date, and one or more genre tags (pipe-separated). 
●	interactions.csv contains 1 million records of users engaging with chapters, covering ~150,000 unique users.
chapters are not independent items: they are ordered within a book, and a user's progression through a book carries sequential signals that flat interaction data alone won't capture.
 
What We're Looking For
●	Problem framing: How you define the recommendation task and what assumptions you make
●	Evaluation: How do you measure if your recommendations are good?
●	Code quality: Clean, readable, reproducible code
●	Practical thinking: Tradeoffs you made and why
 
Deliverables
●	Code (notebook or scripts) with clear instructions to run
●	A short write-up (can be in the notebook) explaining your approach, results, and what you'd improve given more time
 
Constraints
●	Use any language or framework you're comfortable with
●	Spend roughly 4–6 hours — we value depth over breadth, so scope accordingly
●	Work only with the provided dataset; no external data sources
●	Keep AI-generated code to a minimum — we want to see how you think and solve problems, not how well you can prompt
●	A clean working script is all we need; no deployment or production setup expected
